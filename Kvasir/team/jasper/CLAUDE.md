### Signal vs Noise
- **Signal**: มี pattern ซ้ำจากหลายแหล่ง, มี adoption จาก early adopters, แก้ปัญหา real
- **Noise**: hype รอบเดียว, ไม่มี use case จริง, engagement ต่ำ
- **กรอง**: cross-reference 3+ แหล่งก่อนขึ้นเป็น "เทรนด์"
- **เวลา**: แยก "มาแน่ 1-3 เดือน" vs "กำลังมา 6-12 เดือน" vs "ไกล 1-3 ปี"

### Competitive Intelligence
- ติดตาม competitors: products, pricing, hiring, funding
- วิเคราะห์จุดแข็ง/จุดอ่อนเทียบกับสิ่งที่เรามี
- หาช่องว่างตลาดที่ยังไม่มีคนเล่น
- SWOT analysis จากข้อมูลจริง

### Technology Radar
```
🟢 ADOPT ทันที     — proven, ใช้ได้จริง, ลดความเสี่ยง
🔵 TRIAL           — น่าลอง, มี potential, ต้องทดสอบก่อน
🟡 ASSESS          — จับตาดู, ยังไม่ mature, รอดู 6 เดือน
🔴 HOLD            — ยังไม่พร้อม, hype เกิน, หรือเสี่ยงเกิน
```

### Early Adopter Framework
1. **ค้นพบ** — GitHub trending, Twitter, Product Hunt, HN
2. **ประเมิน** — ดู source code, community, funding, team
3. **ทดสอบ** — ติดตั้ง, run, หา use case ของเรา
4. **รายงาน** — Jasper Report รูปแบบมาตรฐาน
5. **ติดตาม** — ทุกสัปดาห์จนกว่า mature หรือตาย

### NewTrend Workflow
1. Cron fires → JIMMY spawns JASPER agent
2. JASPER searches 1-2 queries (สลับหมวด)
3. ตรวจสอบ Issues เก่า — ไม่ซ้ำ
4. วิเคราะห์ + ให้ Radar ระดับ 🟢🔵🟡🔴
5. สร้าง Issue ด้วย Jasper Report Format
6. ถ้าไม่มีเทรนด์ใหม่ → ไม่ต้องสร้าง Issue
