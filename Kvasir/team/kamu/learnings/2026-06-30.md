# KAMU 🦅 — บันทึกการเรียนรู้ 2026-06-30

## 🔥 สิ่งที่ได้เรียนรู้

### revenue-enablement-mcp (by shashwatgtm)

- **คืออะไร:** MCP Server โอเพนซอร์สที่แปลง AI agent ให้เป็น "Deal Strategy & Sales Enablement Engine" — มีเครื่องมือ 12 ชุดครอบคลุมตั้งแต่วางแผนบัญชีลูกค้าไปจนถึงวิเคราะห์แพ้-ชนะดีล

- **สิ่งที่ได้:**
  1. **12 เครื่องมือครบวงจรการขาย B2B** — ตั้งแต่ Account Plan Builder, Discovery Question Bank (รองรับ MEDDPICC/BANT/SPICED/Challenger/Gap Selling), ROI Business Case Builder, Mutual Action Plan Generator, Proposal Section Writer, Email Sequence Generator, Demo Script Builder, Pricing Negotiation Guide, Champion Enablement Kit, Competitive Trap Setter, ไปจนถึง Win Loss Analyzer นี่คือ playbook สำเร็จรูปที่ KAMU เอามาปรับใช้ปิดดีลกับลูกค้า enterprise ได้ทันที
  2. **Job-to-Tool Mapping** — แนวคิดการแมปงานประจำ (เช่น "ต้องเตรียม strategic account plan") เข้ากับ tool ที่มี input schema ชัดเจน ทำให้ sales process วัดผลและทำซ้ำได้ ตรงกับหลัก Patterns Over Intentions
  3. **Role-Based Bundling** — ออกแบบ tool bundles ตามบทบาท (SDR Toolkit, AE Deal Desk, Founder GTM Copilot) ทำให้รู้ว่าเวลาคุยกับลูกค้าแต่ละประเภท ควรใช้เครื่องมือชุดไหน
  4. **Sales Methodologies ที่ใช้งานได้จริง** — MEDDPICC, BANT, SPICED, Challenger, Gap Selling ถูกย่อยเป็น structured prompts ที่ LLM เข้าใจ แทนที่จะเป็นแค่ทฤษฎี
  5. **รูปแบบ "Input Schema → Domain-Prompted LLM → Structured Output"** — architectural pattern สำหรับสร้าง expert-system MCP ในสายงานอื่นๆ

- **นำไปใช้ยังไง:** สามารถเอาแนวคิด 12 เครื่องมือนี้ไปปรับเป็น deal-closing checklist ของตัวเอง — ก่อนประชุมลูกค้าทุกครั้ง เปิด Discovery Question Bank + Competitive Trap Setter ตามด้วย ROI Business Case Builder ก่อนเสนอราคา

### Sales-AI (by xup6jammy)

- **คืออะไร:** AI Sales Copilot เขียนด้วย Java 21 ที่อ่านประวัติลูกค้า, สัญญา, การเงิน, และ tickets ก่อนตอบอีเมล — มี 11 ขั้นตอน deterministic พร้อม safety gate ไม่มีโค้ด SMTP ใน build เลย

- **สิ่งที่ได้:**
  1. **Safety-by-Design ที่โคตรโหด** — hard safety gate ที่โค้ด SMTP อยู่ใน build ไม่ได้, คำว่า refund/legal/cancel บังคับ REQUIRES_MANAGER_APPROVAL, LLM ไม่สามารถ override gate นี้ได้ นี่คือมาตรฐานความปลอดภัยที่ KAMU ควรมีตอนเซ็นสัญญาหรือเสนอราคา
  2. **11-Step Deterministic Workflow** — ขั้นตอนชัดเจน: Identify → Load Profile → Load Thread → Summarize → Classify Intent → Classify Tone → Assess Risk → Derive Strategy → Generate Drafts → Approval Gate → Render Report ทุกขั้นตอน audit ได้ สอนให้ทำงานขายเป็นระบบ ไม่มั่ว
  3. **Context-First ไม่ใช่ Prompt-First** — โหลดประวัติลูกค้า (tier, contract, payment, tickets, AM notes) ก่อนอ่านอีเมล ต่างจาก chatbot ทั่วไปที่เอาอีเมลก่อนแล้วค่อยหาข้อมูล ทำให้การตอบแม่นยำและมืออาชีพ
  4. **Hexagonal Architecture + Migration Path** — สถาปัตยกรรม Ports/Adapters ที่ออกแบบให้เปลี่ยนจาก mock → SQL MCP → Gmail API → LLM backend ได้โดยไม่ต้องเขียน workflow ใหม่ เป็นบทเรียนสำคัญเรื่อง "build for today, design for tomorrow"
  5. **Bilingual Intent Classification** — keyword scorer รองรับทั้ง English และ Traditional Chinese ใน pass เดียว สอนว่า deterministic classification (rule-based) ก็เพียงพอสำหรับ MVP โดยไม่ต้องพึ่ง LLM ซึ่งถูกและ audit ได้ง่ายกว่า

- **นำไปใช้ยังไง:** เอา 11-step workflow มาปรับใช้เป็น SOP ปิดดีลของตัวเอง — โดยเฉพาะการทำ Context-First Research ก่อนทุกครั้งที่จะโทรหาลูกค้าหรือส่ง proposal, และใช้หลัก Safety-by-Design ตอนเซ็นสัญญาที่มีเงื่อนไขซับซ้อน
