### shixinzhang/tiktok-viral-hooks
- **คืออะไร:** คลังเปิดที่รวบรวมการถอดโครงสร้างวิดีโอ TikTok ที่มียอดวิวสูง วิเคราะห์แบบเจาะลึกว่าทำไมถึงดัง — มีมากกว่า 80 รูปแบบ hook พร้อมตัวอย่างจริงและ template ที่ copy ได้ทันที
- **สิ่งที่ได้:**
  - **80+ Hook Patterns Taxonomy:** รู้จัก hook ทุกประเภทที่ใช้ใน TikTok จริง — Curiosity Gap, Challenge/Test, Question-Based, Contrast, Revelation, Emotional Confession, Direct Address, Promise of Transformation, Narrative Immersive, Scarcity/Urgency และอื่นๆ — แต่ละแบบมีหลาย variant ย่อย Curiosity Gap ใช้บ่อยที่สุด
  - **3-Second Rule เจาะลึก:** 3 วินาทีแรกคือทุกสิ่ง ทุก breakdown ระบุ exact opener และชื่อ pattern ของมัน
  - **Retention Architecture:** จุดรักษาผู้ชมอยู่ที่ ~7, ~14, ~21 วินาที — ต้องมี pattern interrupt ในจังหวะเหล่านี้
  - **2-Sentence Script Template:** ทุกวิดีโอจะถูกย่อยเป็น template 2 ประโยคที่เอา topic เราใส่แทนได้ทันที
  - **Mind Map Architecture:** มีแผนภาพโครงสร้างสคริปต์ทั้งหมด ทำให้เห็น flow การเล่าเรื่องและ retention
  - **Top Video Case Studies:** วิดีโออันดับ 1 (76.3M views) ใช้ Contrast Escalation — อันดับ 2 (50.6M) ใช้ Childhood Secret Code Setup — ดูแนวโน้มว่าอะไรเวิร์ค
  - **Niche Distribution Insight:** Entertainment 65 เรื่อง, Beauty 14, Food 12, Finance 10 — niche ที่ content เน้นขายของเองมีน้อยกว่า entertainment มาก
- **นำไปใช้ยังไง:** เลือก hook patterns ที่เข้ากับเนื้อหา GemLogin มาใช้ เช่น Curiosity Gap กับฟีเจอร์ที่คนไม่รู้, Contrast กับ Before/After การทำงานแบบ manual vs automation, Authority Statement กับโปรไฟล์ของทีม

### vyralcontent/content-skills
- **คืออะไร:** ชุด "Agent Skills" โอเพนซอร์สจาก Vyral Content ที่วิเคราะห์ TikTok, Reels, Shorts กว่า 200,000 ตัว แล้วกลั่นเป็น 7 frameworks/system สำหรับสร้าง content แบบมืออาชีพ — รองรับ Claude Code และ Cursor
- **สิ่งที่ได้:**
  - **Three-Layer Hook System:** hook ไม่ได้มีแค่คำพูด — ต้องมี Visual + Verbal + On-Screen Text พร้อมกันใน 1-3 วินาทีแรก การออกแบบทั้ง 3 ชั้นพร้อมกันเพิ่มพลัง hook อย่างมหาศาล
  - **Core Script Structure:** Hook → Escalation → Payoff → CTA — โครงสร้างที่พิสูจน์แล้วสำหรับ high-retention content ใช้ได้กับทุก platform
  - **Platform-Specific Algorithm Logic:**
    - TikTok → completion rate, trending sound timing, duets/stitches
    - Reels → Sends Per Reach (ส่งต่อ) คือ metric ที่พาไปถึงคนนอก follower
    - YouTube Shorts → VVSA, Engaged Views, การ reroute คนดูเข้าสู่ long-form
  - **Content Mining System:** วิธีไม่ให้หัวโล่ง — mine ไอเดียจาก audience comments, Reddit, search autocomplete, competitor outliers, และ repurposing engine
  - **Caption Layer = Distribution Critical:** มีระบบ captions, on-screen text, hashtags, CTAs, pinned comment ที่ทำให้ content ถูกแจกจ่ายและถูก save
  - **Anti-Patterns:** มีการระบุสิ่งที่ทำแล้วฆ่า retention หรือโดน algorithm Penalty อย่างชัดเจน
  - **Trial Reels:** ใช้ test content ก่อนลง public ช่วยเซฟทรัพยากร
- **นำไปใช้ยังไง:** นำ Three-Layer Hook System มาใช้กับ content ของ GemLogin — ภาพหน้าจอ (Visual) + Voiceover อธิบาย (Verbal) + ข้อความเด่นๆ (Text) — และใช้โครงสร้าง Hook → Escalation → Payoff → CTA สำหรับทุกวิดีโอ

### rediumvex/social-media-caption-generator-claude
- **คืออะไร:** Claude Code Skill สำหรับสร้าง caption โพสต์โซเชียลมีเดีย โดยปรับตาม algorithm signal ของแต่ละ platform — สร้างจากสคริปต์, screenshot, หรือ competitor post ที่ผู้ใช้ป้อนเข้าไป
- **สิ่งที่ได้:**
  - **Platform Algorithm Signals ที่แตกต่างกัน:**
    - Instagram Reels → DM shares (ไม่ใช่ Likes)
    - Instagram Carousel → Saves
    - TikTok → Completion rate 70%+
    - Threads → Replies
    - Facebook Page → 40-80 ตัวอักษร
    - YouTube Shorts → Title SEO + retention
  - **30+ Caption Formulas:** แบ่งตามเป้าหมาย — reach, engagement, sales, growth — แต่ละสูตรมีโครงสร้างเฉพาะ
  - **Hook Library ใน Caption:** caption ทุกอันต้องมี hook type, formula name, CTA type, และ signal target
  - **Length Rules ต่อ Platform:** IG Reels 80-200 chars, FB Page 40-80 chars, TikTok 100-300 chars — ยาวเกินหรือสั้นเกินไปเสียโอกาส
  - **Anti-Patterns & Shadow Ban Triggers:** มีรายการสิ่งที่ห้ามใช้ — engagement bait ("comment yes"), trigger words ที่ทำให้ reach ลด 70%, generic hashtags (#fyp #viral)
  - **Power Words Library:** คำที่กระตุ้น urgency, trust, exclusivity, emotion — ใช้เพิ่มอัตราการตอบสนอง
  - **FB Trigger Word Replacement:** คำว่า "FREE" → "complimentary" และอื่นๆ เพราะ FB ปรับลด reach คำพวกนี้
  - **Signal Hierarchy:** shares > saves > comments > watch time > likes — เรียงลำดับความสำคัญได้ชัดเจน
- **นำไปใช้ยังไง:** นำ platform-specific caption formulas และ anti-patterns มาใช้เขียนคำบรรยาย content ของ GemLogin โดยเฉพาะ TikTok (เน้น completion rate) และ Facebook Page (สั้น 40-80 ตัวอักษร หลีกเลี่ยง trigger words) — และเพิ่ม Power Words ใน CTA

### tsensei/OpenReels
- **คืออะไร:** Pipeline โอเพนซอร์สภาษา TypeScript ที่เปลี่ยน topic เป็นวิดีโอสั้นแนวตั้ง (Short/Reel/TikTok) เสร็จพร้อมลง — ทำงานอัตโนมัติ 8 ขั้นตอน ตั้งแต่ research ถึง render MP4
- **สิ่งที่ได้:**
  - **8-Stage Pipeline สำหรับ Video Creation:**
    1. Research — หาข้อมูล grounding ป้องกัน hallucination
    2. Script — เขียนสคริปต์พร้อม scene breakdown, visual direction, emotional arc
    3. Voiceover — TTS พร้อม word-level timestamps สำหรับ karaoke captions
    4. Visuals — สร้างภาพ AI, AI video clips, stock footage ที่ vision-verified
    5. Music — AI-generated background หรือ royalty-free tracks
    6. Captions — Spring-animated 3-state captions, 7 styles, karaoke highlighting
    7. Assembly — รวมทุกอย่างเป็น MP4 แนวตั้งด้วย Remotion
    8. Critique — AI critic ให้คะแนน output ถ้าไม่ถึงเกณฑ์ pipeline ทำงานซ้ำ
  - **14 Visual Archetypes:** editorial_caricature, warm_narrative, studio_realism, infographic, anime_illustration, comic_book, gothic_fantasy, surreal_dreamscape, cinematic_documentary, moody_cinematic, bold_illustration และอื่นๆ — เปลี่ยนอารมณ์ content ได้ด้วยการเปลี่ยน archetype
  - **Cost Transparency:** แสดง cost breakdown ก่อนเสียเงินจริง ทำให้วางแผน budget ได้
  - **Dry-Run Mode:** ดู DirectorScore JSON ก่อน generate จริง — ลดความเสี่ยงเสียทรัพยากร useless
  - **Creative Direction Brief:** ป้อน markdown brief เพื่อ steer ทิศทาง AI — ใช้คุมโทน content ให้ตรงกับแบรนด์
  - **Karaoke-Style Captions:** คำที่พูดจะถูก highlight ตาม word-level timestamps — เทคนิค retention ที่ได้ผลดีบน TikTok
  - **CI/CD Ready:** `--yes` flag สำหรับ Docker/CI environments
  - **Multi-Provider Flexibility:** เลือก LLM/TTS/Image/Video/Music provider ได้ตามสะดวก — Kokoro TTS ฟรีไม่ต้องใช้ API key
- **นำไปใช้ยังไง:** ศึกษา pipeline เพื่อออกแบบ content creation workflow สำหรับ GemLogin โดยเฉพาะ — ใช้แนวคิดเรื่อง archetype ในการกำหนด visual identity ของแบรนด์, ใช้โครงสร้าง script breakdown เพื่อวางแผน content ล่วงหน้า, และใช้เทคนิค karaoke captions เพิ่ม retention ให้ TikTok content เกี่ยวกับ GemLogin

### ธีม content ที่เป็นไปได้:

| Direct Address | "สำหรับคนที่เบื่อทำฟาร์มแมนนวล..." |
| Relatable Vulnerability | "ผมเสียเวลา setup farm ไป 3 วัน เพราะไม่รู้ tool นี้" |

### โครงสร้างวิดีโอ 1 เรื่อง (จาก content-skills):
1. **Hook (3 วิ):** Curiosity Gap + Visual (ภาพ interface) + Text ("5,000 tasks/day?")
2. **Escalation (7-14 วิ):** อธิบาย pain point ที่ทวีขึ้น + On-screen text ประกอบ
3. **Payoff (14-21 วิ):** โชว์ GemLogin ทำงาน — automation ทำงานเสร็จใน 30 วิ
4. **CTA (21-24 วิ):** Power word + คำกระตุ้น ("อยากลอง? ลิงก์ใน bio")

### แนวทางลงแพลตฟอร์ม (จาก social-media-caption-generator):
- **TikTok:** Caption 100-300 chars, เน้น completion rate, ใช้ karaoke caption (จาก OpenReels)
- **Facebook Page:** Caption 40-80 chars, หลีกเลี่ยง trigger words ("FREE" → "complimentary"), ใช้ discussion question
- **Instagram Reels:** เน้น DM shares ใน caption, สั้นกระชับ 80-200 chars

### ขั้นตอนการผลิต content (จาก OpenReels):
1. **Research:** หา pain point จริงจากกลุ่มเป้าหมาย
2. **Script:** วาง scene breakdown พร้อม emotional arc
3. **Voiceover:** อัดเสียงหรือใช้ TTS
4. **Visuals:** จับ screen record จริง + ภาพประกอบ
5. **Captions:** ใส่ word-level karaoke เพื่อ retention
6. **Critique:** ให้คะแนน content ก่อนลงจริง — content ไม่ดี = ไม่ลง
