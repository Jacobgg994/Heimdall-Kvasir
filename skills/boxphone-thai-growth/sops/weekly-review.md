# SOP — Weekly Review (Monday Morning · 60 min)

Single meeting per week that replaces 15 dashboards. Run every Monday 09:00–10:00.

## Attendees

- CEO (driver)
- Lead VA (notes + asks for help)
- Optional: content editor, CDS account manager once headcount expands

## Pre-meeting (15 min before, the VA does this)

Open these tabs in order; have them up on screen when CEO walks in:

1. `/dashboard` — our admin overview
2. `/dashboard/affiliate` — last 7 days CTR by product/post/phone
3. `/dashboard/cms` — pending CDS client deliverables
4. TikTok Shop Affiliate dashboard
5. Shopee Affiliate Centre
6. Lazada Affiliate panel
7. gemphonefarm dashboard (cloud or local)
8. The 90-day plan markdown — current-week section
9. Risk register — review marked items

## Meeting agenda (timed)

### 0–10 min · Numbers

Read aloud, write into the meeting log:

- TikTok Shop revenue (last 7d, cumulative month, target this month)
- Shopee + Lazada commission (last 7d, cumulative month)
- CDS MRR booked + collected (this month)
- LINE OA setups closed (this month) + retainer count
- Cumulative cash collected vs. plan
- Fleet: online count / 60, ban events this week

### 10–25 min · What worked

Pick 3 things that worked harder than expected this week:

1. ____
2. ____
3. ____

For each, ask: *can we 2× it this week?* If yes, decide who owns the 2×.

### 25–40 min · What didn't

Pick 3 things that underperformed:

1. ____
2. ____
3. ____

For each, choose one of three actions: **fix · cut · ignore (one-week observation)**. No middle ground — middle ground kills time.

### 40–50 min · Decision triggers

Open the decision-triggers table from the 90-day plan. For each trigger:

- Are we within a week of the trigger date?
- Has the trigger been hit?
- If hit, execute the pre-committed response NOW. No re-debate.

Common triggers to check first:
- Day-14 / Day-30 / Day-45 / Day-60 / Day-90 revenue checkpoints
- CDS client count
- LINE OA setup count
- Fleet health (>50 online required)

### 50–60 min · Calendar for the week

Lock the week ahead:

- Mon: ____
- Tue: ____
- Wed: ____
- Thu: ____
- Fri: ____
- Weekend if anything: ____

Write into a shared doc with owner names. VA carries it forward in daily check-ins.

## After-meeting (5 min)

CEO sends a 4-line summary to all team via LINE:

```
Week N · Mon update
✅ Wins: ___
⚠️  Issues: ___
🎯 This week: ___
👀 Watch: ___
```

This forces public commitment + gives every team member same context for the week.

## Don't do in this meeting

- Detailed strategy debate (separate quarterly call)
- New plays / new ideas (write them in a parking-lot doc; review monthly)
- Personnel performance discussion (separate 1-1)
- Tech architecture (separate engineering sync)

## What success looks like

- Meeting ends on time, every time
- Every action item has an owner + deadline
- Numbers trend up week-over-week
- No surprise mid-week ("we forgot to..." doesn't happen)

## Failure mode

If the Monday review feels rushed, it means the 90-day plan stopped being read.
If it feels like nothing changes week to week, you're not pulling the decision-trigger lever.
If new ideas keep gate-crashing it, you don't have a parking-lot doc.

## Quarterly variant

Once per quarter, expand to 3 hours and:

- Review the full 90-day plan vs. reality
- Update the risk register
- Re-price CDS tiers if AI tokens shifted
- Decide the next quarter's plays
- Hire / fire decisions
